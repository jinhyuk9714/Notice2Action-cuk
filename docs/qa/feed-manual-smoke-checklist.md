# Feed Manual Smoke Checklist

## Purpose
- 개인화 피드의 정렬 품질을 실제 학부생 시나리오 기준으로 확인한다.
- 새 기능 추가 전후에 상위 10개 공지가 설득력 있게 정렬되는지 반복 검증한다.
- 자동 테스트로 다 잡히지 않는 `오탐`, `다른 대상 공지` 과상승, 약한 reason 문구 문제를 수동으로 확인한다.

## Environment
- 저장소: `Notice2Action-clean`
- 메인 화면: `#/feed`
- 확인 범위: 게시판 칩은 `전체` 상태를 기본으로 둔다.
- 각 프로필마다 상위 10개 공지만 확인한다.

## Representative Profiles

### 1. 컴퓨터정보공학부 / 1학년 / 신입생
- 선호 게시판: `학사`, `일반`
- 키워드: `학생증`, `학번`, `I-DESIGN`
- 상단에 기대하는 공지 유형:
  - 신입생 수강신청
  - I-DESIGN 수강신청
  - 신·편입생 학번조회
  - 학생증 관련 공지

### 2. 컴퓨터정보공학부 / 3학년 / 재학생
- 선호 게시판: `학사`, `취창업`
- 키워드: `수강신청`, `취업`
- 상단에 기대하는 공지 유형:
  - 수강과목 취소
  - 취업학생 출결
  - 취창업 프로그램
  - 군 e-러닝

### 3. 학과 미지정 / 4학년 / 졸업예정자
- 선호 게시판: `학사`, `장학`
- 키워드: `졸업`, `예비 졸업사정`, `트랙`
- 상단에 기대하는 공지 유형:
  - 졸업대상자 공지
  - 예비 졸업사정
  - 융복합트랙 확인
  - 장학 관련 마감 공지

### 4. 컴퓨터정보공학부 / 2학년 / 재학생
- 선호 게시판: `학사`, `장학`
- 키워드: `장학금`, `국가장학금`
- 상단에 기대하는 공지 유형:
  - 국가장학금
  - 인문100년

### 5. 학과 미지정 / 3학년 / 재학생

- 키워드: `부전공`, `복수전공`
- 상단에 기대하는 공지 유형:

  - 복수전공 관련 공지
  - 학사 절차 공지

## Top 10 Checkpoints
- 상위 10개 안에 명백한 오탐이 1개를 넘지 않는가
- `다른 대상 공지` 배지가 붙은 공지가 same-profile matched actionable notice보다 위에 오지 않는가
- keyword-only 정보성 공지가 matched actionable notice를 밀어내지 않는가
- 선호 게시판 공지가 같은 tier의 비선호 게시판 공지보다 앞서거나 최소 같은 수준으로 유지되는가
- `행동 필요` 공지가 정보성 공지보다 충분히 위에 오는가
- reason 문구가 실제 상위 노출 이유와 일치하는가
- `선호 게시판` reason이 붙어야 할 공지에 누락되지 않는가
- `학생증 관련`, `학번 관련`, `졸업 관련` 같은 키워드 reason이 과하게 남발되지 않는가
- 최신성 배지(`최근 등록`, `이번 주 등록`)가 강한 profile/actionability 신호를 덮지 않는가
- 게시판 배지와 reason 배지가 서로 충돌하지 않는가

## Failure Types
- `오탐`: 상위에 오면 안 되는 공지가 상단에 노출됨
- `대상 오판`: `다른 대상 공지`가 과하게 높거나, 반대로 관련 공지가 낮음
- `키워드 과민`: 키워드만 맞아서 정보성 공지가 과하게 상승
- `선호 게시판 과민`: 선호 게시판 때문에 relevance가 약한 공지가 과하게 상승
- `reason 불일치`: 카드에 적힌 이유가 실제 정렬 이유로 납득되지 않음

## Recording Format
아래 형식으로 프로필별로 1회 기록한다.

```md
### profile: cs_junior_enrolled
- top 3 notices:
  1. ...
  2. ...
  3. ...
- findings:
  - [ ] 명백한 오탐 없음
  - [ ] 다른 대상 공지가 matched actionable notice보다 아래
  - [ ] keyword-only informational notice 과상승 없음
  - [ ] preferred board positive notice ordering 자연스러움
  - [ ] reason 문구 납득 가능
- notes:
  - ...
```

## Escalation Rule
- 같은 유형의 실패가 2개 이상 반복되면 `NoticeFeedService` 점수/우선순위 조정 대상으로 승격한다.
- 단일 공지 1건만 어색한 경우는 fixture에 `expectedMustAppearAbove`를 추가할지 먼저 판단한다.
- exact rank를 강제하지 않는다. `high > mid > low`와 `must appear above`만 고정한다.
